Welcome to Meet!

Introduction

What does Meet API do? - Meet is an online medical consultation API. What that means is that it allows users to sign up as patients or doctors. If they sign up as patients, they can create a complaint(a complaint is basically a set of symptoms), possible issues get suggested based on the symptoms in the complaint and then get a list of doctors that specialize in providing treatment in that field. The doctors are rated based on reviews and ratings from patients they have had consultations with in the past, the patients can then pick their doctor of choice based on those ratings and availability. After picking a doctor, the doctor schedules appointments with the patient for further discussions and consultation.

Who can use Meet API? – Meet API is available to individuals seeking to test the functionality of the API. It is also available to developers who wants to integrate with the API (Frontend and Backend developers)

Now that we know what the Meet API can do and who can use the API, let’s dive right into the requirements and steps to follow in order to use the API!

Create Account

What you need to do before you can use the Meet API is create an account. Creating an account automatically generates a token for you which is needed to authorize all requests that you will be making to the API endpoints

  • create account to test the API

    If you are willing to interact with the Meet API just to test its functionality, you can only do so by making calls to its endpoints using a service like Postman or using language libraries like Python Requests, JavaScript JQuery e.t.c. This is so because Meet is just an API and for that reason it doesn’t have an interface that allows users to interact with it directly from the frontend.

    *Note: request examples in this documentation are made using python requests library

    To create an account;
    The fields required are:
    • email(required)
    • first_name(required)
    • last_name(required)
    • username(required)
    • password(required)
    • password2(required) - it must match password 1
    • category(required) – you can sign up either as a patient or a doctor
    • phone_number(not required)

    make a POST request to :

    http://127.0.0.1:8000/api/users/register/

    Sample request

    body = {

    'email': 'test@gmail.com',
    'first_name': 'test',
    'last_name': 'user',
    'username': 'tester',
    'password': 'testuserpassword',
    'password2': 'testuserpassword',
    'category': 'PT' (PT for patient),
    'phone_number': 'testusernumber'(do not use country code;enter the number as a string)

    }
    headers = {

    'Content-Type': 'application/json'

    }
    url = 'http://127.0.0.1:8000/api/users/register/'
    request = requests.post(url, headers=headers, json=body)
    response_data = request.json()

    Sample response

    {


    'respone': 'successfully registered user',
    'email': 'test@gmail.com',
    'username': 'tester',
    'category': 'PT',
    'first_name': 'test',
    'last_name': 'user',
    'phone_number': 'testusernumber,
    'token': 'your generated token'

    }

    'Content-Type': 'application/json'

Endpoints

  • Complaint endpoints
    • Complaint CRUD endpoints

      • Create:
        The fields required are

        • sex(required) - Choice field and can either be 'male' or 'female'
        • year_of_birth(required) - 4 digits and It must be a year lesser than the current year
        • age_group(required) - Choice field: preteen(PT), teenager(TN), adult(AD), old adult(OAD)

          • make sure that the abbreviations are what you enter for the age_group field

        To add symptoms, add a query parameter called 'symptoms' to your url and populate it with IDs of the symptoms you want to add in the format shown below

        After entering values for the fields, make a POST request to:

        http://127.0.0.1:8000/api/booking/complaint/create?symptoms=ID1_ID2_ID3

        Sample request

        body = {

        'sex': 'male',
        'year_of_birth': 2003,
        'age_group': 'AD',

        }
        headers = {

        'Authorization': 'Token {your generated token}',
        'Content-Type': 'application/json'

        }
        url = 'http://127.0.0.1:8000/api/booking/complaint/create/'
        query_params = {

        'symptoms': '68_66'

        }
        request = requests.post(url, params=query_params, headers=headers, json=body)
        response_data = request.json()

        Sample response

        {

  • User management endpoints
    • User CRUD endpoints

      *The user creation endpoint has been covered in one of the sections above so we won’t go into that

      When a user creates an account as a doctor or an account for integration, they get a profile that holds details about the field(s) they specialize in, the age group of patients they treat(Choice field: preteen(PT), teenager(TN), adult(AD), old adult(OAD)) as specified by them, the meets they have, their reviews and star rating.

      • specialization
      • patient_type(Choice field: preteen(PT), teenager(TN), adult(AD), old adult(OAD))
